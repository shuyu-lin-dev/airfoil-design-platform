# 翼型气动与强度智能设计平台

这个上下文定义翼型气动与强度智能设计平台的核心领域语言，帮助需求、接口、测试和后续智能体工具封装使用同一套术语。

## Language

**CST 参数**:
用于描述二维翼型几何形状的一组 12 个有序数值参数，顺序为前 6 个上表面系数、后 6 个下表面系数。
_Avoid_: 翼型参数、几何参数

**工况参数**:
描述气动或强度预测所处流动条件的参数集合，当前包含 `mach` 和 `angle_of_attack`。
_Avoid_: 环境参数、飞行参数

**评估条件**:
预测或优化评估时保持固定的外部条件，第一版等同于 **工况参数**。
_Avoid_: 优化变量

**优化变量**:
优化算法允许改变的输入变量，第一版气动优化改变 **CST 参数**，强度优化改变 **结构优化变量**，耦合优化改变二者。
_Avoid_: 输入参数

**优化比例**:
优化目标或结果的相对变化比例，使用小数表达且范围为 `0 < ratio <= 1`。
_Avoid_: 百分数字符串、百分数

**结构设计参数**:
描述三维机翼中的梁、蒙皮和肋等结构几何，第一版包含 `spar_thickness`、`skin_thickness` 和 `rib_spacing`。
_Avoid_: 强度参数、结构参数

**结构优化变量**:
强度优化第一版允许改变的结构设计参数子集，包含 `spar_thickness` 和 `rib_spacing`。
_Avoid_: 结构设计参数

**机翼平面参数**:
描述三维机翼基础尺度的参数集合，第一版包含可选的 `span` 和 `chord`。
_Avoid_: 翼型参数、几何参数

**材料属性**:
参与强度和重量预测但不描述结构几何的材料属性，当前包含 `elastic_modulus` 和 `material_density`。
_Avoid_: 材料参数、默认参数

**二维翼型点**:
二维翼型几何曲线上的采样点，包含 `x` 和 `y` 坐标，第一版固定 200 个并按闭合轮廓顺序排列。
_Avoid_: xy 点、二维坐标点

**Cp 分布**:
二维翼型点上的压力系数分布，第一版与 **二维翼型点** 按索引一一对应。
_Avoid_: 气动分布、压力分布

**教学控制点**:
教学界面用于表达中弧线或厚度分布趋势的归一化平面点，不是严格的 Bezier 或 B-spline 控制点。
_Avoid_: Bezier 控制点、B-spline 控制点

**三维机翼几何点**:
三维机翼几何上的采样点，包含 `x`、`y` 和 `z` 坐标。
_Avoid_: 三维翼型点、场数据点

**设计样本**:
一次预测或优化评估中的具体翼型/机翼参数组合及其派生几何。
_Avoid_: 翼型、样本

**场数据点**:
绑定到空间坐标上的气动场预测结果，例如压力场点或速度场点。
_Avoid_: 几何点

**Artifact**:
后端生成并可被后续接口查询的数据产物，第一版主要指同步写入的本地 HDF5 文件及其同名 JSON sidecar 元信息。
_Avoid_: 文件、结果文件

**耦合适应度**:
衡量气动强度耦合优化目标的标量，第一版定义为 `lift_drag_ratio / weight`。
_Avoid_: 综合评分、优化分数

**重量**:
机翼受重力作用产生的力，API 字段为 `weight`，单位 N。
_Avoid_: mass、质量

**结果元信息**:
描述一次生成、预测或优化结果来源可信度的响应字段，长期包含 `is_stub`，可选包含 `model_version`。
_Avoid_: 调试字段、临时字段

## Relationships

- 一个 **CST 参数** 集合生成一组 **二维翼型点**。
- **CST 参数** 的顺序是契约的一部分：`upper[0..5]` 在前，`lower[0..5]` 在后。
- 第一版 **二维翼型点** 的前 100 个是上表面从前缘到后缘，后 100 个是下表面从后缘回到前缘，不重复第一个点。
- **Cp 分布** 与 **二维翼型点** 按索引一一对应。
- **教学控制点** 的 `x` 是归一化弦向位置，范围为 `0 <= x <= 1`。
- **工况参数** 会传入优化算法用于模型评估，但属于 **评估条件**，不是 **优化变量**。
- **优化比例** 中的 `0.10` 表示 10%，不接受 `"10%"` 或 `10`。
- **CST 参数** 定义二维翼型截面；强度预测、强度优化和耦合优化的对象是三维机翼。
- **机翼平面参数** 可以由用户覆盖，但第一版优化器不会改变它。
- 三维机翼生成、强度预测、强度优化和耦合优化都接收完整的 **结构设计参数**。
- 第一版 **结构优化变量** 只包含 `spar_thickness` 和 `rib_spacing`，不包含 `skin_thickness`。
- **材料属性** 可以由用户覆盖，但第一版优化器不会改变它。
- **耦合适应度** 随升阻比增大而增大，随重量增大而减小。
- 第一版 API 暴露 **重量**，不暴露质量；若未来模型输出质量，需要先转换为 **重量**。
- **结果元信息** 是长期 API 契约；第一版占位结果返回 `is_stub: true` 和 `model_version: "stub-v0"`。
- 一个 **三维机翼几何点** 使用坐标系 `x = 弦向`、`y = 厚度方向`、`z = 展向`。
- 不同 **设计样本** 的空间坐标点可能不同。
- **场数据点** 不是几何定义本身，而是压力、速度等预测结果在空间位置上的取值。
- 一个 **Artifact** 可以保存某个 **设计样本** 的坐标、压力场、速度场等大数组数据，并通过 REST 返回元信息。
- 第一版 **Artifact** 元信息落盘为同名 JSON sidecar，不只存在内存中。
- 第一版 **Artifact** 只保存单个 **设计样本**，不保存优化候选历史。
- 第一版 **Artifact** 采用同步写入、异步语义：返回前写完文件，但保留 `pending / ready / failed` 状态契约。

## Example dialogue

> **Dev:** “三维机翼生成接口返回的是压力场点吗？”
> **Domain expert:** “不是。它返回的是 **三维机翼几何点**。压力场和速度场属于 **场数据点**，第一版通过 **Artifact** 保存。”
>
> **Dev:** “`skin_thickness` 是材料属性吗？”
> **Domain expert:** “不是。它描述蒙皮几何，属于 **结构设计参数**；**材料属性** 只描述材料本身。”
>
> **Dev:** “那强度优化会改 `skin_thickness` 吗？”
> **Domain expert:** “第一版不会。`skin_thickness` 是 **结构设计参数**，但不是 **结构优化变量**。”
>
> **Dev:** “耦合优化里两个方案怎么比较？”
> **Domain expert:** “第一版比较 **耦合适应度**，也就是 `lift_drag_ratio / weight`，升阻比越高、重量越低越好。”
>
> **Dev:** “优化器能不能把攻角调小来提高升阻比？”
> **Domain expert:** “不能。`angle_of_attack` 属于 **评估条件**，会传入模型计算结果，但不是 **优化变量**。”
>
> **Dev:** “`cst_params[7]` 是上表面还是下表面？”
> **Domain expert:** “下表面。**CST 参数** 前 6 个是上表面系数，后 6 个是下表面系数。”
>
> **Dev:** “第一版气动预测返回 `artifact_id` 时，HDF5 一定写完了吗？”
> **Domain expert:** “是的。第一版 **Artifact** 同步写入，返回时通常已经是 `ready`，只是契约为未来异步实现保留状态字段。”
>
> **Dev:** “服务重启后还能查询 artifact 吗？”
> **Domain expert:** “可以。第一版 **Artifact** 元信息写在同名 JSON sidecar 里，不只存在内存 registry。”
>
> **Dev:** “压力场和速度场能共用一套全局默认坐标吗？”
> **Domain expert:** “不能。不同 **设计样本** 的坐标可能不同，**Artifact** 必须保存自己的 `/coordinates`。”
>
> **Dev:** “强度耦合优化要不要把每一代候选样本都写进 HDF5？”
> **Domain expert:** “第一版不要。**Artifact** 只保存单个 **设计样本**，优化接口只返回原始/优化后的参数和指标。”
>
> **Dev:** “模型如果输出 kg，API 也返回 kg 吗？”
> **Domain expert:** “不。API 只返回 **重量** `weight`，单位 N；质量需要在模型适配层先转换。”
>
> **Dev:** “Cp 的第 120 个点对应哪一个几何点？”
> **Domain expert:** “对应第 120 个 **二维翼型点**。第一版 **Cp 分布** 和 **二维翼型点** 按索引一一对应。”
>
> **Dev:** “教学控制点是不是 Bezier 控制点？”
> **Domain expert:** “不是。**教学控制点** 只是教学界面的归一化趋势点，第一版不承诺 Bezier 或 B-spline 算法。”
>
> **Dev:** “目标提升 10% 时传 `10` 还是 `0.10`？”
> **Domain expert:** “传 `0.10`。**优化比例** 一律用小数表达，不能用百分数字符串。”
>
> **Dev:** “接入真实模型后可以删掉 `is_stub` 吗？”
> **Domain expert:** “不删。`is_stub` 属于 **结果元信息**，真实模型结果把它设为 `false`，并填写模型版本。”
>
> **Dev:** “强度预测只靠二维 CST 参数够吗？”
> **Domain expert:** “不够。**CST 参数** 只定义二维翼型截面；强度预测的对象是三维机翼，需要 **机翼平面参数** 或配置默认尺度。”

## Flagged ambiguities

- “三维机翼生成点数”曾同时写成 1000 和 10000。已解决：后端 MVP 固定返回 1000 个 **三维机翼几何点**。
- “强度参数/结构参数/材料参数”曾混用。已解决：第一版使用 **结构设计参数** 表示梁、蒙皮、肋等结构几何，使用 **材料属性** 表示材料本身。
- “适应度函数”曾未定义。已解决：第一版 **耦合适应度** 固定为 `lift_drag_ratio / weight`。
- “优化后的工况参数”曾暗示优化器可以改变工况。已解决：**工况参数** 是固定 **评估条件**，会传入优化算法但不是 **优化变量**。
- “12 个 CST 参数顺序”曾未定义。已解决：前 6 个为上表面系数，后 6 个为下表面系数。
- “Artifact 异步写入”曾可能暗示第一版需要任务队列。已解决：第一版采用同步写入、异步语义。
- “Artifact 元信息”曾可能只存在内存 registry。已解决：第一版使用同名 JSON sidecar 落盘。
- “场数据坐标”曾可能被理解为全局默认坐标。已解决：每个 **Artifact** 保存对应 **设计样本** 自己的 `/coordinates`。
- “优化 artifact”曾可能扩展为保存所有候选历史。已解决：第一版 **Artifact** 只保存单个 **设计样本**。
- “质量/重量”曾混用。已解决：第一版 API 只暴露 **重量** `weight`，单位 N，不暴露 `mass`。
- “二维翼型点顺序”曾未定义。已解决：前 100 个为上表面前缘到后缘，后 100 个为下表面后缘到前缘，不重复第一个点；**Cp 分布** 按索引对齐。
- “控制点”曾可能被理解为 Bezier/B-spline 控制点。已解决：第一版使用 **教学控制点**，只表示归一化趋势点。
- “优化幅度”曾可能被理解为百分数字符串或整数百分数。已解决：第一版 **优化比例** 统一用小数表达，范围为 `0 < ratio <= 1`。
- “is_stub”曾可能被理解为临时字段。已解决：`is_stub` 属于长期 **结果元信息**，第一版为 `true`，真实模型结果为 `false`。
- “翼型/机翼”曾混用。已解决：**CST 参数** 定义二维翼型截面；强度预测、强度优化和耦合优化的对象是三维机翼，并使用 **机翼平面参数** 表示基础尺度。
- “结构设计参数”和“结构优化变量”曾混用。已解决：第一版 **结构优化变量** 只包含 `spar_thickness` 和 `rib_spacing`，不包含 `skin_thickness`。
