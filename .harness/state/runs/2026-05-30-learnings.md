# 学习记录 — 2026-05-30

## 违规：任务完成协议 6 步遗漏

**触发场景**：用户要求"使用 harness 开启新任务进行项目自检"，执行了删除 `backend/backend/` + 环境自检 + pytest 验证后，直接宣告完成，未执行 git commit 及后续收尾步骤。

**遗漏的 6 步**（9 步协议中）：

| 步骤 | 遗漏项 | 根因 |
|------|--------|------|
| 2. 提交 | 未 git add/commit | 把"验证通过"等同于"任务完成"，忽略了提交是协议硬性步骤 |
| 3. 状态 | 未在 tasks.yaml 建任务条目 | 自检类维护活动未纳入任务模型，属于流程设计空白 |
| 5. 决策 | 未检查 DECISIONS.md | 没有养成"每次改动后检查是否需要记录决策"的习惯 |
| 6. 学习 | 未写 learnings | 同上，没有养成习惯 |
| 7. 遥测 | 未填 telemetry.yaml | 遥测模板只有骨架，从未被实际填充过 |
| 9. 自检 | 未跑 session-exit-checklist | 把 checklist 当成"参考"而非"硬性步骤" |

## 根因分析

1. **"验证通过 = 完成"的错误心智模型**：pytest 通过后心理上就认为任务结束了，忽视了 harness 协议中验证只是第 1 步，后面还有 8 步。
2. **自检类活动无任务模板**：tasks.yaml 只建模了功能开发任务（T000-T015），项目自检、bug 修复、目录清理等维护活动没有对应的任务条目模板。这导致自检活动游离在任务模型之外。
3. **遥测从未被实际使用**：telemetry.yaml 自创建以来 runs 列表为空，说明这个工件在当前工作流中形同虚设。

## 改进措施

- 今后任何改动，验证通过后立即检查 9 步协议清单，不跳过任何一步。
- 对于非功能开发类活动（自检、修复、清理），要么在 tasks.yaml 中建临时任务条目，要么在 PROGRESS.md 中明确标注为"非任务维护活动"并说明不适用 9 步协议的原因。
- 遥测要么实际使用，要么从协议中降级为可选。
