# I/O 卫生规则

工具输出回灌进上下文是 agent 循环退化的首要原因。长输出不处理会在几轮内耗尽上下文窗口，导致推理质量下降。以下规则在每次工具调用后自动适用。

## 截断

- 命令输出（pytest、uvicorn、pip、shell）默认只保留尾部 2000 行。超出部分丢弃，不在上下文中展示。
- 若尾部 2000 行不足以定位问题（如失败发生在输出前段），agent 可以主动 grep 或 tail 更多行。

## 抽帧

- Python traceback：只保留最后一条异常类型、消息、报错帧（含文件名和行号）和失败测试名。丢弃中间的库内部帧和不相关帧。
- 编译错误：只保留第一条错误的位置和消息，丢弃后续级联错误。

## 落盘

- 输出 >5MB 的命令结果必须写入文件（`.harness/state/working/`），上下文只保留文件路径和首尾各 20 行摘要。
- agent 按需读取文件片段，不将完整大文件装入上下文。

## 错误分支清理

- 若 agent 沿错误方向跑了两轮以上，用对话 rewind 删除错误分支，避免错误推理继续占据窗口。
- 重试前先写一句当前阻塞点和下一步方向到 working state，防止 rewind 后丢失诊断信息。
