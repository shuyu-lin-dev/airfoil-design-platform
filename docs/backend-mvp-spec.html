API 契约

理解 Pydantic BaseModel 与 FastAPI 的数据流转

一句话：BaseModel 是 Pydantic 提供的数据类，你用它定义"这个接口收什么字段、回什么字段"， 然后像普通 Python 类一样 return 响应模型(字段=值, ...)，FastAPI 自动帮你转成 JSON 发给客户端。

第 1 步 — 定义数据结构
写一个 Python 类，继承 BaseModel，声明字段和类型。这就是"数据契约"。 
from pydantic import BaseModel

class Point2D(BaseModel):
    x: float
    y: float

class Airfoil2DResponse(BaseModel):
    points: list[Point2D]
    is_stub: bool
    model_version: str
第 2 步 — 路由函数中 return
像普通 Python 类一样创建实例，return 出去。这不是 JSON，是一个 Python 对象。 
@router.post("/geometry/airfoil-2d")
def generate_airfoil_2d(
    req: Airfoil2DRequest
) -> Airfoil2DResponse:
    pts = [Point2D(x=..., y=...) for _ in range(200)]
    return Airfoil2DResponse(
        points=pts,
        is_stub=True,
        model_version="stub-v0"
    )
第 3 步 — FastAPI 自动转 JSON
你 return 的 Python 对象被 FastAPI 自动序列化为下面的 JSON 字符串，通过 HTTP 返回给客户端。 
// 客户端收到的 HTTP Response Body:
{
  "points": [
    {"x":1.0, "y":0.0},
    {"x":0.998, "y":0.005},
    ... 共 200 个点 ...
  ],
  "is_stub": true,
  "model_version": "stub-v0"
}
简单理解：
• BaseModel = 一个 Python 类，定义了"这个数据结构里有哪些字段、各自什么类型"。作用类似 C 语言的 struct。
• return Airfoil2DResponse(points=..., is_stub=True, ...) = 创建一个该类的实例（Python 对象），和 return SomeClass(arg=val) 一样。
• FastAPI 看到你 return 了一个 BaseModel 实例，自动调用 .model_dump() 转成 dict，再 json.dumps() 变成 JSON 字符串返回。
• 同理，客户端发来的 JSON 字符串，FastAPI 自动解析成你声明的 BaseModel 实例（如 req: Airfoil2DRequest），你可以直接 req.cst_params 访问字段。
方法路径输入（HTTP Request JSON）输出（HTTP Response JSON）领域
GET/health —status, version Health
POST/geometry/airfoil-2d cst_params: float[12] points: {x,y}[200], is_stub, model_version Geo
POST/geometry/wing-3d cst_params[12], wing_planform, structure_design points: {x,y,z}[1000], is_stub, model_version Geo
POST/aerodynamics/predict cst_params[12], condition{mach, aoa} lift_drag_ratio, cp_distribution[200], field_artifact, is_stub Aero
POST/structure/predict cst_params[12], condition, wing_planform, structure_design, material_properties max_stress (Pa), weight (N), is_stub Struct
POST/optimization/aerodynamic cst_params[12], condition, target_improvement_ratio original{...}, optimized{...}, actual_improvement_ratio, is_stub Opt
POST/optimization/structural cst_params[12], condition, wing_planform, structure_design, material_properties, target_reduction_ratio original{...weight}, optimized{...weight}, actual_reduction_ratio, is_stub Opt
POST/optimization/coupled cst_params[12], condition, wing_planform, structure_design, material_properties, target_improvement_ratio original{...fitness}, optimized{...fitness}, actual_improvement_ratio, is_stub Opt

所有 POST 响应统一携带 is_stub: true 和 model_version: "stub-v0"。 比例字段统一使用小数（10% = 0.10，范围 0 < ratio ≤ 1）。

▶ Geo POST /geometry/airfoil-2d — 二维翼型生成
1. Pydantic 请求模型 (Python)
from pydantic import BaseModel, Field

class Airfoil2DRequest(BaseModel):
    cst_params: list[float] = Field(
        min_length=12, max_length=12
    )
HTTP Request Body (JSON)
{
  "cst_params": [0.1,0.2,0.3,0.1,0.2,0.3,
                  -0.1,-0.2,-0.3,-0.1,-0.2,-0.3]
}
2. 各层函数与返回值 (Python)
# ═══ core/geometry.py — 纯领域逻辑 ═══
# 返回：普通 Python 数据（list[dict]）
def generate_airfoil_2d_points(
    cst_params: list[float]
) -> list[dict]:
    points = []
    for i in range(200):
        # ... 占位计算 ...
        points.append({"x": x_val, "y": y_val})
    return points  # ← list[dict]，不是 Pydantic

# ═══ services/geometry_service.py — 业务编排 ═══
# 返回：普通 Python 数据（透传，也可以加日志/校验）
def generate_2d(
    cst_params: list[float]
) -> list[dict]:
    return generate_airfoil_2d_points(cst_params)
    # ← 同样返回 list[dict]，不是 Pydantic

# ═══ api/geometry.py — 路由层 ═══
# 返回：Pydantic 响应模型 ← 只有这层返回 BaseModel！
@router.post("/geometry/airfoil-2d")
async def generate_airfoil_2d(
    req: Airfoil2DRequest
) -> Airfoil2DResponse:
    raw = geometry_service.generate_2d(req.cst_params)
    # raw 是 list[dict]，需转成 Pydantic 对象列表
    pts = [Point2D(**d) for d in raw]
    return Airfoil2DResponse(
        points=pts,
        is_stub=True,
        model_version="stub-v0"
    )  # ← 返回 Pydantic 实例，FastAPI 自动转 JSON
规则：只有 api/ 路由函数需要返回 Pydantic BaseModel（FastAPI 据此生成 JSON）。 services/ 和 core/ 返回普通 Python 数据（dict / list / dataclass / 基本类型）， 不依赖 FastAPI，方便单独测试和复用。
3. Pydantic 响应模型 (Python)
class Point2D(BaseModel):
    x: float
    y: float

class Airfoil2DResponse(BaseModel):
    points: list[Point2D]  # 200 个
    is_stub: bool
    model_version: str
HTTP Response Body (JSON)
{
  "points": [
    {"x":1.0,"y":0.0},
    {"x":0.998,"y":0.005},
    ...200 个点...
    {"x":1.0,"y":0.0}
  ],
  "is_stub": true,
  "model_version": "stub-v0"
}
约定：points 按闭合翼型轮廓顺序排列。points[0:100] 为上表面（前缘→后缘），points[100:200] 为下表面（后缘→前缘）。不额外重复第一个点，前端如需闭合自行连接首尾。
▶ Aero POST /aerodynamics/predict — 气动预测
1. Pydantic 请求模型 (Python)
class Condition(BaseModel):
    mach: float
    angle_of_attack: float

class AeroPredictRequest(BaseModel):
    cst_params: list[float] = Field(
        min_length=12, max_length=12
    )
    condition: Condition
HTTP Request Body (JSON)
{
  "cst_params": [0.1,0.2,...],
  "condition": {
    "mach": 0.5,
    "angle_of_attack": 3.0
  }
}
2. 各层函数与返回值
# ═══ core/ + services/ — 返回普通 Python 数据 ═══
# aerodynamic_service.predict() 返回一个简单对象：
#   svc.ld_ratio      → float
#   svc.cp             → list[dict]  (200 个 {x,y,cp})
#   svc.artifact_meta  → dict        (artifact 元信息)
# 这些都是普通 Python 数据，不是 Pydantic BaseModel

# ═══ api/ — 路由层，包装成 Pydantic 响应 ═══
@router.post("/aerodynamics/predict")
async def predict_aerodynamics(
    req: AeroPredictRequest
) -> AeroPredictResponse:
    svc = aerodynamic_service.predict(req)
    # 把普通数据包装成 Pydantic 模型返回
    return AeroPredictResponse(
        lift_drag_ratio=svc.ld_ratio,
        cp_distribution=[CpPoint(**d) for d in svc.cp],
        field_artifact=FieldArtifactMeta(**svc.artifact_meta),
        is_stub=True,
        model_version="stub-v0"
    )  # ← 只有这层返回 Pydantic BaseModel
3. Pydantic 响应模型 (Python)
class CpPoint(BaseModel):
    x: float; y: float; cp: float

class FieldArtifactMeta(BaseModel):
    artifact_id: str; status: str
    format: str; path: str
    datasets: dict[str,str]

class AeroPredictResponse(BaseModel):
    lift_drag_ratio: float
    cp_distribution: list[CpPoint]  # 200
    field_artifact: FieldArtifactMeta
    is_stub: bool
    model_version: str
HTTP Response Body (JSON)
{
  "lift_drag_ratio": 15.3,
  "cp_distribution": [
    {"x":1.0,"y":0.0,"cp":-0.5},
    ...200 个...
  ],
  "field_artifact": {
    "artifact_id":"abc-123",
    "status":"ready","format":"hdf5",
    "path":"runtime_artifacts/aerodynamic/abc-123.h5",
    "datasets":{
      "coordinates":"/coordinates",
      "pressure_field":"/fields/pressure",
      "velocity_field":"/fields/velocity"
    }
  },
  "is_stub":true,
  "model_version":"stub-v0"
}
压力场和速度场不直接通过 JSON 返回。预测完成后，后端将 1000 个三维坐标点 + 压力场 (1000 个 scalar, Pa) + 速度场 (1000 个 scalar, m/s) 写入 HDF5 文件，同时生成同名 JSON sidecar。JSON 响应中只包含 HDF5 artifact 的元信息（artifact_id / path / datasets 列表），客户端需要场数据时通过 GET /artifacts/{artifact_id} 查询路径后自行读取 HDF5 文件。
▶ Struct POST /structure/predict — 强度预测
Pydantic 请求模型
class WingPlanform(BaseModel):
    span: float | None = None
    chord: float | None = None

class StructureDesign(BaseModel):
    spar_thickness: float
    skin_thickness: float
    rib_spacing: float

class MaterialProperties(BaseModel):
    elastic_modulus: float | None = None
    material_density: float | None = None

class StructurePredictRequest(BaseModel):
    cst_params: list[float] = Field(
        min_length=12, max_length=12)
    condition: Condition
    wing_planform: WingPlanform
    structure_design: StructureDesign
    material_properties: MaterialProperties
Pydantic 响应模型
class StructurePredictResponse(BaseModel):
    max_stress: float    # 单位: Pa
    weight: float        # 单位: N
    is_stub: bool
    model_version: str
HTTP Response Body (JSON)
{
  "max_stress": 250000000.0,
  "weight": 9800.0,
  "is_stub": true,
  "model_version": "stub-v0"
}
关键约束
weight (N) vs mass (kg)：
API 只暴露 weight（重量，单位 N）。
若未来模型输出质量 mass (kg)，
模型适配层内部转换：
weight = mass × 9.80665
第一版占位模型直接输出 weight (N)。
可选字段默认值：
span / chord / elastic_modulus / material_density 均可选。
默认值统一由 config/settings.py 管理。
▶ Opt POST /optimization/aerodynamic — 单一气动优化
Pydantic 请求模型
class AeroOptRequest(BaseModel):
    cst_params: list[float] = Field(
        min_length=12, max_length=12)
    condition: Condition
    target_improvement_ratio: float = Field(
        gt=0, le=1
    )
Pydantic 响应模型
class AeroOptSnapshot(BaseModel):
    cst_params: list[float]
    condition: Condition
    lift_drag_ratio: float

class AeroOptResponse(BaseModel):
    original: AeroOptSnapshot
    optimized: AeroOptSnapshot
    actual_improvement_ratio: float
    is_stub: bool
    model_version: str
约束
只优化 cst_params，不改变 condition。
target_improvement_ratio 使用小数（10% = 0.10），范围 (0, 1]。
▶ Opt POST /optimization/structural — 单一强度优化
优化变量（可改变）
spar_thickness梁厚度，单位 m
rib_spacing肋间距，单位 m
不改变的字段
cst_params不涉及气动外形
condition工况是评估条件

核心数据模型

所有字段均通过 Pydantic 定义，自动校验类型和约束。HTTP 线路上以 JSON 传输。

CST CST 参数（翼型外形参数化）

cst_paramsfloat[12]12 个 CST 系数，控制翼型几何外形
前 6 个 = 上表面系数（upper_0..upper_5）
后 6 个 = 下表面系数（lower_0..lower_5）
约束：必须正好 12 个；第一版不强制物理范围

Cond 工况参数（来流条件）

machfloat马赫数（无量纲），飞行速度与当地音速之比
angle_of_attackfloat攻角（单位 degree），翼型弦线与来流方向的夹角
工况是预测和优化的评估条件。优化算法接收工况参数以调用模型计算升阻比/应力/重量，但不改变工况参数。

Plane 机翼平面参数

spanfloat?翼展（单位 m），机翼左右两端之间的展向长度
chordfloat?弦长（单位 m），机翼剖面前缘到后缘的弦向长度
cst_params 只定义二维翼型截面形状；三维机翼需要 span（翼展）和 chord（弦长）确定空间尺度。可选字段，未传时使用 config/settings.py 中的默认值。

Struct 结构设计参数

spar_thicknessfloat梁厚度（单位 m），机翼主承力梁的厚度
skin_thicknessfloat蒙皮厚度（单位 m），机翼表面蒙皮的厚度
rib_spacingfloat肋间距（单位 m），相邻翼肋之间的展向距离
⚠ 结构设计参数 ≠ 结构优化变量：第一版优化只改变 spar_thickness 和 rib_spacing，skin_thickness 参与强度计算但优化过程不改变它。

Mat 材料属性

elastic_modulusfloat?弹性模量（单位 Pa），材料抵抗弹性变形的能力，默认 70 GPa（铝合金）
material_densityfloat?材料密度（单位 kg/m³），默认 2700（铝合金）
材料属性参与强度和重量预测但不作为结构几何。可选字段，默认值由 config/settings.py 管理。业务代码不得在 service/model/core 中硬编码。

Pt 点类型 & 坐标系

坐标系：x = 弦向（前缘→后缘），y = 厚度方向，z = 展向
Point2D{x,y}二维翼型点，200 个/翼型
Point3D{x,y,z}三维机翼点，1000 个/机翼
CpPoint{x,y,cp}Cp（压力系数）分布点，无量纲，与翼型点一一对应
PressurePoint{x,y,z,pressure}压力场点，单位 Pa（走 HDF5，不走 JSON）
VelocityPoint{x,y,z,velocity}速度场点，单位 m/s，第一版标量（不走 JSON）

Meta 结果元信息（所有响应共用）

is_stubbool是否为占位结果。第一版固定 true，未来真实模型改为 false
model_versionstr模型版本标识。第一版固定 "stub-v0"，未来如 "aero-ld-v1.2"
这两个字段长期保留，是结果可信度边界的一部分。不要移除 is_stub。

Ratio 比例字段规范

target_improvement_ratiofloat目标提升比例（气动/耦合优化用），范围 (0,1]，如 10% → 0.10
target_reduction_ratiofloat目标降低比例（强度优化用），范围 (0,1]，如 10% → 0.10
actual_improvement_ratiofloat实际达到的提升比例
actual_reduction_ratiofloat实际达到的降低比例
统一使用小数表达，不使用百分数字符串。三种优化（气动/强度/耦合）的停止条件字段不能混用。

Artifact 策略：JSON vs HDF5 分工

通过 REST JSON 返回（小数据 & 元信息）

  • 标量预测结果（升阻比、max_stress、weight）
  • 200 个二维翼型点
  • 200 个 Cp 分布点
  • 优化前后参数对比
  • Artifact 元信息（artifact_id, path, datasets 列表）

通过 HDF5 文件存储（全场数据）

  • 压力场 — 1000 个 scalar，单位 Pa
  • 速度场 — 1000 个 scalar，单位 m/s（标量）
  • 三维坐标 — 1000×3 (x,y,z)
  • 后续更大的三维场数据

HDF5 文件内部布局 & 同名 JSON Sidecar

HDF5 内部 Dataset 结构

/coordinates       shape: (1000, 3)   columns: [x, y, z]
/fields/pressure   shape: (1000,)     unit: Pa
/fields/velocity   shape: (1000,)     unit: m/s

/fields/pressure[i] 和 /fields/velocity[i] 对应 /coordinates[i]。 每个 artifact 保存自己的坐标数据，不依赖全局默认坐标。

同名 JSON Sidecar（元信息文件，必须存在）

{
  "artifact_id": "abc-123",
  "format": "hdf5",
  "path": "runtime_artifacts/aerodynamic/abc-123.h5",
  "status": "ready",
  "datasets": {
    "coordinates": "/coordinates",
    "pressure_field": "/fields/pressure",
    "velocity_field": "/fields/velocity"
  }
}
输出目录：backend/runtime_artifacts/aerodynamic/（由 .gitignore 忽略）。
写入策略：第一版同步写入（调用预测接口后 artifact 立即可查询，status=ready）。
范围：第一版 artifact 只保存单个设计样本的全场数据，不保存优化迭代历史。
查询：GET /artifacts/{artifact_id} 读取 JSON sidecar 返回元信息，不存数据库。

关键职责边界

contracts/ vs models/contracts = API 数据契约 (Pydantic)；models = ML 模型适配层。名字相似，职责完全不同。
结构设计参数 vs 优化变量skin_thickness 是结构设计参数但不是优化变量。设计参数有 3 个，优化变量只有 2 个。
weight vs massAPI 只暴露 weight (N)，不暴露 mass (kg)。若未来模型输出质量，模型适配层须内部转换。
教学 vs 工程教学接口 (/teaching/*) 和工程接口 (/geometry/*, /aerodynamics/*, /structure/*, /optimization/*) 分属不同边界。
JSON vs HDF5小数据 (≤200 点) 走 JSON；全场数据 (1000+ 点) 走 HDF5。
config 事实源所有默认值（材料属性、重力加速度 g、artifact 根目录、stub 模式）统一从 config/settings.py 读取，不得散落硬编码。

优化算法真实遗传算法、贝叶斯优化、多目标优化
可视化前端实时渲染 / 后端生成图片 / 导出文件

智能体调度LangGraph + DeepSeek，ReAct 模式
异步化长耗时任务可能改为异步任务队列

风险点（需测试保护）

cst_params 必须正好 12 个，顺序固定（前 6 = 上表面系数，后 6 = 下表面系数）
工况参数单位固定：mach 无量纲，angle_of_attack 为 degree
三维机翼强度相关接口必须显式处理 wing_planform 或使用配置默认机翼尺度
结构设计参数 ≠ 材料属性，skin_thickness 属于结构设计参数而非材料属性
结构设计参数 ≠ 结构优化变量：第一版仅优化 spar_thickness + rib_spacing
weight 统一表示重量 (N)，API 不暴露 mass；若未来模型输出 kg 须在适配层转换
默认材料属性、重力加速度 g、artifact 根目录、stub 模式统一从 config/settings.py 读取
速度场第一版是标量 velocity，不是 u/v/w 三分量
压力场和速度场不能误走 JSON，必须写入 HDF5 artifact
所有生成/预测/优化响应必须长期保留 is_stub + model_version
HDF5 artifact 必须有同名 JSON sidecar 保存元信息，不能只依赖内存 registry
HDF5 artifact 必须保存设计样本自己的 /coordinates，压力场和速度场不得依赖全局默认坐标
第一版 artifact 只保存单个设计样本的全场数据，不保存优化候选历史
坐标系固定：x = 弦向，y = 厚度方向，z = 展向
耦合适应度第一版固定为 fitness = lift_drag_ratio / weight
所有比例字段使用小数表达，输入范围 0 < ratio ≤ 1（10% = 0.10）
气动/强度/耦合优化的停止条件字段不能混用（improvement_ratio vs reduction_ratio）
教学接口和工程接口不能混在同一边界里
